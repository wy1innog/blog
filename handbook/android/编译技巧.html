    # Android10关闭dex优化

    Android Q中按照以往方法修改frameworks/base/core/目录，然后在根目录make framework编译出的framework.jar，直接adb push到手机system/framework/目录下，重启机器会出现无法启动现象，循环报出boot image异常，导致system server无法启动。

    # 解决方法

    修改build/make/core目录下的board_config.mk和dex_preopt_config.mk文件。

    1. board_config.mk文件，修改：

      WITH_DEXPREOPT := ture
      1

      为

      WITH_DEXPREOPT := false
      1

    2. dex_preopt_config.mk文件，修改：

      WITH_DEXPREOPT_BOOT_IMG_AND_SYSTEM_SERVER_ONLY ?= ture
      1

      为

      WITH_DEXPREOPT_BOOT_IMG_AND_SYSTEM_SERVER_ONLY ?= false
      1

    3. 注释如下逻辑：

      ifeq (,$(filter eng, $(TARGET_BUILD_VARIANT)))
  ifneq (true,$(WITH_DEXPREOPT))
    ifneq (true,$(WITH_DEXPREOPT_BOOT_IMG_AND_SYSTEM_SERVER_ONLY))
      $(call pretty-error, DEXPREOPT must be enabled for user and userdebug builds)
    endif
  endif
endif

    # 刷入手机验证

    编译完成之后的产物目录

    android-10.0.0_r41\out\target\product\blueline\system\framework\framework.jar

    目录下有一个framework.jar(这个jar就是框架的代码) 里面都是dex，如果想看代码，可以去这个目录下看，就好像平时看第三方jar一样，导入as即可。

    android-10.0.0r41\out\soong.intermediates\frameworks\base\framework\androidcommon\jarjar\framework.jar

    开始刷入。

    // 获取root权限
adb root  

// 关闭检测功能
adb disable-verity 

// 关闭检测之后需要重启机器
adb reboot 

重启机器成功之后：

// 再次获取root权限
adb root  

// 重启挂载system分区为可读写分区
adb remount  

// 推送framework.jar

adb root && adb remount

adb push framework.jar /system/framework

adb reboot

    # 总结

    1. 方便后面修改模块和验证，建议从开始就关闭dex优化。
    2. 修改单个模块之后，去模块的目录下看看是否有mk文件，如果有在目录下mm编译即可。
    3. 刷入手机可能会失败，先准备好代码分支对应的官方刷机包，随时刷回去官方的，保证我们下次刷入自己编译的顺利。

    # 常用命令

    # 编译命令

    • make snod: 打包out/target/product/xxx/system的内容并生成新的system.img文件
    • make vnod: 从构建系统中快速编译出vendor.img
    • make pnod: 从构建系统中快速编译出product.img
    • make senod: 从构建系统中快速编译出system_ext.img
    • make onod:从构建系统中快速编译出odm.img

    # 清除命令

    • m clobber: 清除所有编译缓存，相当于rm -rf out/
    • m clean: 清除编译缓存，相当于rm -rf out/target/product/[product_name]
    • m installclean: 清除所有二进制文件

    # 跳转命令

    • croot: 可以快速回到android源码根目录
    • godir [file name]: 定位到包含指定文件的目录下，file name采用模糊匹配，如果有多个目录包含指定的file name，则需要手动选择

    # 查找命令

    • jgrep: 在所有的java文件上执行grep指令
    • cgrep: 在所有的c/c++文件上执行grep指令
    • rgrep: 在所有的res/*.xml文件上执行grep指令
    • ggrep: 在所有的Gradle文件上执行grep指令
    • mangrep: 在所有的androidmanifest.xml文件上执行grep指令
    • mgrep: 在所有的Makefiles和*.bp文件上执行grep指令
    • segrep: 在所有的sepolicy文件上执行grep指令
    • sgrep: 在所有的文件上执行grep指令

    # 其他指令

    • printconfig: 显示当前编译环境的配置信息
    • allmod: 显示AOSP所有的module
    • pathmod [module name]: 显示module name所在的路径
    • refreshmod: 刷新module列表
    • gomod [module name]: 定位到指定的module name目录下
