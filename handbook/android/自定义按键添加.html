    # 修改流程

    # 一、驱动通过GPIO连接的按键

    此类按键采用GPIO来连接，通过监测其中断来判断按键的动作，需要根据具体硬件设计在项目对应的dts文件配置gpio_keys节点。

    底层驱动主要修改以下两个文件上报键值。

    1. 修改dts文件

      // bsp/kernel/kernel4.14/arch/arm64/boot/dts/sprd/rt5-zx01-overlay.dts
gpio-keys {
				compatible = "gpio-keys";
				key-volumedown {
								label = "Volume Down Key";
								linux,code = <KEY_VOLUMEDOWN>;
								gpios = <&ap_gpio 124 GPIO_ACTIVE_LOW>;
								debounce-interval = <2>;
								wakeup-source;
				};

				key-volumeup {
								label = "Volume Up Key";
								linux,code = <KEY_VOLUMEUP>;
								gpios = <&pmic_eic 4 GPIO_ACTIVE_HIGH>;
								debounce-interval = <2>;
								wakeup-source;
				};

				key-power {
								label = "Power Key";
								linux,code = <KEY_POWER>;
								gpios = <&pmic_eic 1 GPIO_ACTIVE_LOW>;
								debounce-interval = <2>;
								wakeup-source;
				};

+				key-hm {
+				        label = "HM Key";/*hm*/
+				        linux,code = <0x253>;
+				        gpios = <&ap_gpio 52 GPIO_ACTIVE_LOW>;
+				        debounce-interval = <2>;
+				        wakeup-source;
+				};

				 key-zdy {
				         label = "ZDY Key";
				         linux,code = <0x254>;/*zdy key linux下的key code，linux下input.h中有定义键值*/
				         gpios = <&ap_gpio 125 GPIO_ACTIVE_LOW>; /* 按键连接的GPIO*/
				         debounce-interval = <2>; /* 按键去抖时间，单位ms，如果出现按键不稳定，可以适当加大*/
				         wakeup-source;
				 };
};

    其中：

    0253 是十六进制数，对应十进制数为595

    1, 0是指按下和弹起的动作

    将input event报的键值转换成具体键盘对应的按键供android上层使用时通过键盘布局文件(*.kl)完成转换的。

    1. 在gpio-keys.kl文件中添加自定义key值

      在kl文件中可以仿照power键添加key值

      这样就可以完成物理按键 kl文件到 KEY_FUNCTION的映射。

      这个kl文件是通过DeviceCommon.mk中编译到系统中，在手机目录的/system/usr/keylayout下可以找到。

      不同平台代码可能存在差异。
